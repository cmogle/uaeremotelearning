# Prompt Library

These prompts are written for the content director to reuse with Codex.

## Content Prompts

### Make copy calmer

> Make the student-facing copy calmer, warmer, and less pressurising. Keep the language concrete, low-jargon, and easy to scan for a stressed learner.

### Reduce reading load

> Reduce the reading load in this section. Shorter sentences, fewer ideas per paragraph, and clearer next actions. Keep the meaning the same.

### Improve accessibility wording

> Rewrite this for learners with additional needs. Reduce abstraction, make support explicit, and keep one action per line where possible.

### Strengthen adult guidance

> Tighten the adult/educator guidance so it is easier to scan on a phone. Keep it practical and avoid pastoral jargon.

### Correct details

> Update the named contacts and links in the affected site variant. Do not change the structure unless needed to keep the page clear.

## Presentation Prompts

### Make the first screen less overwhelming

> Keep the same information, but make the first screen feel calmer and easier to understand for a learner who is already stressed.

### Improve mobile scanning

> Improve mobile scanning in this section. Keep strong hierarchy, good spacing, and large enough tap targets without adding visual clutter.

### Avoid AI-slop design

> Refine the layout so it feels intentional and human-authored. Avoid generic AI-looking cards, decorative noise, or over-styled gradients.

## Structural Prompts

### Split journeys more clearly

> This is a structural request. Explore a clearer separation between student and adult journeys without increasing reading load for students.

### Add a new support route

> This is a structural request. Propose and then implement a new route for [type of user/problem], making sure the existing student-first default stays intact.

### Change root behavior

> This is a structural request. Explain the safest way to change how the root route behaves across the Vercel projects before making the change.

## Review / Revision Prompts

### Ask for another pass

> This is closer, but it still feels too dense. Please do another pass focused on lower cognitive load and stronger scan-ability.

### Reject technical language

> Remove any wording that sounds internal, technical, editorial, or like planning language. The public site should sound human and direct.

### Check both variants

> Before finalizing, check both site variants and confirm there is no copy bleed, wrong contact detail, or audience mismatch.

## Safety Prompt

Use this when you are unsure whether a request is bigger than a normal content change:

> Before you implement this, tell me whether it is content-only, presentation, or structural. If it is structural, explain the risks in plain language first.
